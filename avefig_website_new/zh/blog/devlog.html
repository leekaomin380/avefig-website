    🛠️ 开发工具链选择

    VS Code + Live Server：选择这个组合的原因是简单可靠，适合静态网站开发，实时预览功能大大提高开发效率。

    🎨 设计哲学

    极简主义：参考伯克希尔·哈撒韦的网站风格，专注内容本质，避免视觉干扰，体现专业性和可信度。

    📱 响应式策略

    移动优先：考虑到用户可能在手机上查看教育信息，采用响应式设计确保跨设备体验一致。

    技术债务和改进计划

    当前限制

    • 纯静态网站，无后端数据处理能力
    • 语言切换依赖手动HTML链接
    • 内容更新需要直接修改HTML文件
    • 缺少搜索和筛选功能

    改进计划

    • 短期：引入Markdown内容管理系统
    • 中期：考虑无头CMS或静态站点生成器
    • 长期：根据用户需求决定是否需要后端服务
